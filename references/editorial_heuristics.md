# Editorial Heuristics

This file accumulates cross-run lessons.
The orchestration script appends or refreshes sections here after runs and feedback.

## Current recurring guidance
- Prefer explicit caveats over unsupported certainty.
- Favor at least three high-credibility sources before drafting.
- Keep source-verifier findings as hard blockers when they indicate unsupported major claims.
- If user feedback repeatedly asks for more depth, increase subsection development and nuance.
- If user feedback repeatedly asks for more citations, raise evidence density and citation visibility.
- Default toward publication-ready manuscript depth rather than short article length.
- Require concrete methodology, related-work framing, and explicit limitations in serious drafts.
- A manuscript that is structurally thin or obviously short should not pass as publication-ready.
- Prefer including a modest number of useful figures/tables when they improve readability, especially for timelines, conceptual frameworks, comparisons, or methodological workflows; avoid decorative overuse.
- Prefer primary/full-text support for central claims over metadata-only summaries, and downscope aggressively when only indirect support is available.
- For protocol-style papers, include reproducibility aids such as variable-definition tables, threshold summaries, and explicit preprocessing/event-selection rules.
- Under constrained web/source access, allow protocol/evidence-synthesis drafts to proceed when credible accessible direct support is sufficient, but keep causal claims aggressively downscoped.
